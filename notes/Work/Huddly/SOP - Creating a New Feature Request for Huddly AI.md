### Objective

This SOP outlines the steps to create and document a new feature request for Huddly AI, ensuring clarity and proper communication with the team.

### Key Steps

- Go to Google Drive.
    
- Create a new document.
    
- Name the document starting with the word 'Feature' followed by a descriptive name (3-5 words). For example, 'Feature AI Chat Bot.'
    

  
**2. Document Structure** [0:40](https://www.loom.com/share/4bcf6f1cc7df432c8c33779d08a3f3ce?t=40)

![generated-image-at-00:00:40](https://cdn.loom.com/sessions/picture-in-scripture/4bcf6f1cc7df432c8c33779d08a3f3ce-1.jpg?Policy=eyJTdGF0ZW1lbnQiOlt7IlJlc291cmNlIjoiaHR0cHM6Ly9jZG4ubG9vbS5jb20vc2Vzc2lvbnMvcGljdHVyZS1pbi1zY3JpcHR1cmUvNGJjZjZmMWNjN2RmNDMyYzhjMzM3NzlkMDhhM2YzY2UtMS5qcGciLCJDb25kaXRpb24iOnsiRGF0ZUxlc3NUaGFuIjp7IkFXUzpFcG9jaFRpbWUiOjE3NDMyNDUwMzh9fX1dfQ__&Key-Pair-Id=KQOSYIR44AIC0&Signature=o1Nn6cWfim-OfEMCF17OIEZOrRQU5lj3oKCj98HPSaDQ4NU7JB2EKyTOKL9Z4%7ER7p0-qumbxBAcUHvEST9ZXP9PN0JqaYbJ5l5n42XnJhAszVp0DxcevZEGx77nN18CjdQMgrVpy4yxs0RrZdXrBQNxmjl4%7EYEgzDWjcRAP7s8LVjNNrri1TEqag2IRkzM-7K1c197F02tXiiL%7EDjqQyheuPVUBKpOSd%7Eoz9xZqjqbBbuF6Z66yiU4x9-FgGHny9XsMEXb9O8wmX73WFdW71m7cpb-utfjUOCMN-r-52Y6VGO-x0fzXbRSAjO8-dDiPQwSHS-OewOU5cgtctAzSUvA__)

- In the document, include the following sections:
    
    - **Feature Objective**: Describe the solution to the customer's pain points.
        
    - **Customer Driven Value**: Explain how the feature provides value to the customer.
        
    - **Detailed Description**: Include specifics about look and feel, data types, inputs, and outputs.
        
    - **Definition of Done**: Ensure it aligns with agile principles.
        

  
**3. Save the Document** [3:04](https://www.loom.com/share/4bcf6f1cc7df432c8c33779d08a3f3ce?t=184)

- Ensure the document is saved in the product folder in Google Drive.
    
- Double-check that the document appears in the correct location.

- Send an email to Adam (contractor) to notify him of the new feature request.
    
- Include a message stating 'New feature requests. Please review.'
    

  
**5. Include Review Details** [4:39](https://www.loom.com/share/4bcf6f1cc7df432c8c33779d08a3f3ce?t=279)

- Specify the date and time for the review, which should align with the next sprint backlog (always on Wednesdays).
    
- Example: 'Please have this ready for the backlog review on April 2nd.'

### Cautionary Notes

- Always ensure that the document is saved in the correct folder to avoid confusion.
    
- Double-check the email content for accuracy before sending.
    

### Tips for Efficiency

- Use templates for the document structure to save time.
    
- Set reminders for review dates to ensure timely follow-ups.
